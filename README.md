Skyglow Protocol Version 2 (SGP/2) Specification1. OverviewSGP/2 is a binary, stateful protocol designed for a client-server architecture over a reliable transport (e.g., a TLS socket). It features bidirectional messaging, at-least-once delivery guarantees, and support for AES-256-GCM payload encryption.1.1 General ConventionsByte Order: All multi-byte integers transmitted on the wire are Big-Endian (network byte order).Maximum Payload Size: 256 KB (262,144 bytes). Frames exceeding this size are protocol errors.Strings: Any strings within payloads are UTF-8 encoded and length-prefixed with a big-endian uint16.2. Timing and Keep-AlivesTo maintain a healthy connection, the protocol relies on the following timing thresholds:PING_INTERVAL_SEC (60s): The server will send an S_PING if the client is completely silent for 60 seconds.PONG_TIMEOUT_SEC (15s): If a party sends a Ping and does not receive a Pong within 15 seconds, the connection must be declared dead and closed.CHALLENGE_WINDOW_SEC (300s): The maximum clock skew allowed between the client and server during the login phase.6. Important Client RequirementsDeduplication (Critical Requirement)Because SGP/2 guarantees at-least-once delivery, clients may receive duplicate msg_ids (e.g., if a connection drops before a C_ACK reaches the server).Clients MUST maintain a persisted deduplication cache of recently seen msg_id values (e.g., in SQLite or flat files).Recommended capacity: Keep the last 200 msg_ids. LRU eviction is acceptable.Flow: Before passing a notification to the OS, check the cache. If the msg_id exists: immediately send a C_ACK to the server but do not re-display the notification. If it does not exist, process it, send C_ACK, and add the msg_id to the cache.Collapse KeysThe protocol supports APNS-style collapse keys (COLLAPSE_KEY_LEN = 16 bytes). When a non-zero collapse key is utilized, a new notification will replace any previously queued notification sharing the same collapse key for the target device, regardless of the routing key. An all-zero key means no collapse (independent messages).
